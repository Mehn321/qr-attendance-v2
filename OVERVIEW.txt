QUICK START
================================================================================

1. Install Backend Dependencies:
   cd qr-attendance-v2/backend
   npm install

2. Start Backend Server:
   npm run dev
   (Should say: "Server running on http://localhost:3000")

3. Install Mobile Dependencies (NEW TERMINAL):
   cd qr-attendance-v2
   npm install

4. Start Mobile App:
   npm start

5. Open in Expo Go:
   - Scan QR code with Expo Go app
   - Or press 'a' for Android / 'i' for iOS

6. Test Teacher Login:
   - Tap "Teacher Login"
   - Scan QR: TCHR|TCHR001|Demo Teacher
   - Password: teacher123
   - Success!

KEY FEATURES
================================================================================

✅ 60-Second Cooldown
   - Client-side check (AsyncStorage)
   - Server-side check (database)
   - Countdown timer display
   - Exact message: "Slow down! Please wait 47s before scanning again."

✅ 2FA Authentication
   - Factor 1: Scan teacher QR
   - Factor 2: Enter password
   - JWT token (12-hour expiration)
   - Secure storage (SecureStore)

✅ Section Locking
   - Teacher must select section before scanning
   - Prevents mixed attendance
   - Error if not selected

✅ Automatic Time In/Out
   - First scan: Time In
   - Second scan: Time Out
   - Third+ scan: Reject

✅ Real-Time Dashboard
   - Live statistics
   - Scan logs
   - Auto-refresh every 10 seconds

✅ Advanced Filtering
   - By date (calendar picker)
   - By section (dropdown)
   - By name/ID (search)

================================================================================
COMMON TASKS
================================================================================

Change API URL:
  → Edit: hooks/useApi.ts
  → Line: const API_BASE_URL = '...'

Change app name:
  → Edit: app.json
  → Change: "name": "QR Attendance"

Add more teachers:
  → Add to backend/src/database.ts initializeDatabase()
  → Or use admin dashboard (optional feature)

Change token expiration:
  → Edit: backend/src/routes/teacher.ts
  → Change: { expiresIn: '12h' }

Change database location:
  → Edit: backend/.env
  → Change: DB_PATH=...
